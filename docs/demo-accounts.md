# บัญชีทดสอบ (Demo)

> **เฉพาะสภาพแวดล้อมพัฒนา / ทดสอบ** — ห้ามใช้รหัสเหล่านี้บน production หรือ commit ค่าจริงของ production ลง repo

บัญชีถูกสร้าง/อัปเดตเมื่อรัน `npm run db:seed` (ดู `prisma/seed.ts`)

## บัญชีล็อกอิน (NextAuth)

| ประเภท | อีเมล | รหัสผ่าน |
|--------|--------|----------|
| แอดมิน | `demo-admin@justtrip.local` | `DemoJust_Admin_2026` |
| ผู้จัดทริป | `demo-organizer@justtrip.local` | `DemoJust_Organizer_2026` |
| ไกด์ (ทดสอบเลือกในทริป) | `demo-guide@justtrip.local` | `DemoJust_Guide_2026` |

- แอดมิน: เข้า `/admin`  
- ผู้จัด: เข้า `/organizer`  
- ไกด์: ล็อกอินได้เหมือนผู้จัด (`/organizer`) — บัญชีนี้เปิด **โหมดไกด์** (`isGuide`) แล้ว มีรูปโปรไฟล์และไบโอสั้นให้ทดสอบการ์ดบนหน้าทริป  
  - ผู้จัดคนอื่นเลือกไกด์ได้ในฟอร์มสร้าง/แก้ไขทริป: วาง **รหัสผู้ใช้ (cuid)** ของไกด์หรือค้นด้วยอีเมล `demo-guide@…` แล้วเลือกจากรายการ  
  - รหัสผู้ใช้ดูได้ที่ `/organizer/profile` หลังล็อกอินเป็นไกด์

## ผู้จอง (ไม่มีบัญชีในระบบ)

ผู้จองไม่ได้ล็อกอินด้วยอีเมล/รหัสผ่าน — กรอกจองจากหน้าทริป แล้วใช้ **ลิงก์ดูสถานะ** รูปแบบ `/bookings/<token>` (ได้หลังจองหรือจาก Stripe)

ถ้าต้องการทดสอบ flow จอง ให้ล็อกอินเป็นผู้จัดแล้วสร้างและเผยแพร่ทริปจากแอป (seed ไม่สร้างทริปตัวอย่าง)

## เปลี่ยนรหัสผ่านตอน seed

ตั้งใน `.env` แล้วรัน seed ใหม่:

- `SEED_ADMIN_PASSWORD`
- `SEED_ORGANIZER_PASSWORD`
- `SEED_GUIDE_PASSWORD`

ค่าเริ่มต้นของ seed จะตรงกับตารางด้านบนเมื่อไม่ได้ตั้งตัวแปรเหล่านี้
